# 前端场景分离验证文档

## 问题描述
之前的实现中，"从SLAM获取"按钮和"打地图切换点"按钮共用了同一个地图选择对话框，导致逻辑混乱。

## 解决方案
引入新的全局变量 `g_isMapSwitchPointMode` 来区分两种场景。

## 修改内容

### 1. 新增全局变量
```javascript
let g_selectedFlag = -1;
let g_isMapSwitchPointMode = false;  // 🔴 新增：标记是否为地图切换点模式
```

### 2. 修改 `showMapSelectDialog()` 函数
```javascript
// 修改前：使用 g_selectedFlag === 5 判断
if (g_selectedFlag === 5) {
  confirmBtn.textContent = "确认切换";
} else {
  confirmBtn.textContent = "上传选中的地图";
}

// 修改后：使用专用标志 g_isMapSwitchPointMode
if (g_isMapSwitchPointMode) {
  confirmBtn.textContent = "确认切换";
} else {
  confirmBtn.textContent = "上传选中的地图";
}
```

### 3. 修改 `closeMapSelectDialog()` 函数
```javascript
function closeMapSelectDialog() {
  document.getElementById("mapSelectOverlay").style.display = "none";
  g_selectedMapNumber = null;
  g_isMapSwitchPointMode = false;  // 🔴 重置模式标志
  document.getElementById("confirmUploadBtn").textContent = "上传选中的地图";
}
```

### 4. 修改 `confirmUploadMap()` 函数
```javascript
// 修改前：
if (g_selectedFlag === 5) {
  // 地图切换点逻辑
}

// 修改后：
if (g_isMapSwitchPointMode) {
  // 地图切换点逻辑
}
```

### 5. 修改 `fetchAndMark()` 函数
```javascript
if (g_selectedFlag === 5) {
  g_isMapSwitchPointMode = true;  // 🔴 设置地图切换点模式标志
  setMarkStatus("正在获取地图列表...", false);
  await requestMapFromSlam();
  return;
}
```

## 场景对比

### 场景1: "从SLAM获取"按钮（上传地图场景）

**触发路径**：
```
用户点击"从SLAM获取"按钮
  ↓
requestMapFromSlam()
  ↓
showMapSelectDialog(maps)
  ↓
g_isMapSwitchPointMode = false  ← 默认值，未被设置
  ↓
按钮文本: "上传选中的地图"
  ↓
用户选择地图并点击"上传选中的地图"
  ↓
confirmUploadMap()
  ↓
if (g_isMapSwitchPointMode) → false
  ↓
执行原有上传逻辑（上传地图到FTP，切换显示）
```

**特点**：
- ✅ `g_isMapSwitchPointMode` 保持 `false`
- ✅ 按钮显示"上传选中的地图"
- ✅ 执行上传逻辑，**不等待SLAM重定位**
- ✅ 不触发地图切换TCP请求

### 场景2: "打点"按钮（地图切换点场景）

**触发路径**：
```
用户选择"地图切换点"类型（g_selectedFlag = 5）
  ↓
用户点击"打点"按钮
  ↓
fetchAndMark()
  ↓
if (g_selectedFlag === 5) → true
  ↓
g_isMapSwitchPointMode = true  ← 🔴 设置标志
  ↓
requestMapFromSlam()
  ↓
showMapSelectDialog(maps)
  ↓
if (g_isMapSwitchPointMode) → true
  ↓
按钮文本: "确认切换"
  ↓
用户选择地图并点击"确认切换"
  ↓
confirmUploadMap()
  ↓
if (g_isMapSwitchPointMode) → true
  ↓
执行地图切换逻辑：
  1. 记录地图切换点
  2. 触发SLAM地图切换（TCP请求）
  3. 等待SLAM重定位完成（轮询状态）
  4. 显示成功消息
```

**特点**：
- ✅ `g_isMapSwitchPointMode` 被设置为 `true`
- ✅ 按钮显示"确认切换"
- ✅ 执行地图切换逻辑，**等待SLAM重定位**
- ✅ 触发地图切换TCP请求到SLAM端

## 关键改进点

### 改进1: 场景隔离
- **问题**: 之前使用 `g_selectedFlag === 5` 判断，但"从SLAM获取"按钮调用时 `g_selectedFlag` 可能仍然是5
- **解决**: 使用独立的 `g_isMapSwitchPointMode` 标志，只在 `fetchAndMark()` 中设置

### 改进2: 状态重置
- **问题**: 关闭对话框后标志未重置，可能影响下次打开
- **解决**: 在 `closeMapSelectDialog()` 中重置 `g_isMapSwitchPointMode = false`

### 改进3: 按钮文本动态化
- **问题**: 按钮文本固定，用户不清楚当前操作
- **解决**: 根据场景动态设置按钮文本（"上传选中的地图" vs "确认切换"）

## 总结

通过引入 `g_isMapSwitchPointMode` 标志，成功将两种场景的逻辑分离：

| 场景 | 触发方式 | 标志状态 | 按钮文本 | 执行逻辑 |
|------|---------|---------|---------|---------|
| 上传地图 | "从SLAM获取"按钮 | `false` | "上传选中的地图" | 上传到FTP，不等待SLAM |
| 地图切换点 | "打点"按钮（类型=5） | `true` | "确认切换" | 触发SLAM切换，等待重定位 |

✅ **修改完成，两种场景逻辑已完全分离！**
