# 地图切换机制完整分析报告

## 📋 执行摘要

本报告详细分析了基于ROS的多地图SLAM系统的启动流程和地图切换机制。系统通过TCP通信接收地图切换请求，支持多地图间的无缝切换和坐标转换。

## 🔄 地图切换机制详解

### 切换触发条件

**方式1: TCP请求触发（主要方式）**

**监听端口**: 6051

**请求数据结构**:
```cpp
struct req_frame
{
    unsigned long frame_type;   // 目标地图ID（1=map1, 2=map2, 3=map3...）
    unsigned long seq;          // 请求序列号
    float x;                    // 当前坐标X
    float y;                    // 当前坐标Y
    float yaw;                  // 当前航向角（弧度）
};
```

**示例TCP请求**:
```python
# 切换到map2，当前位置为(1.5, 2.0)，航向角0.5弧度
request = req_frame()
request.frame_type = 2      # 目标地图ID
request.seq = 12345         # 序列号
request.x = 1.5             # 当前X坐标
request.y = 2.0             # 当前Y坐标
request.yaw = 0.5           # 当前航向角

# 发送到 192.168.2.100:6051
```

**方式2: 手动调用（测试用）**

可以直接调用 `launchMap` 函数（需要修改代码）

### 切换流程详解

#### 步骤1: 接收TCP请求

**代码位置**: `handleTcpData()` 函数

```cpp
req_frame request;
ssize_t recv_len = recv(client_fd, &request, sizeof(request), 0);

ROS_WARN("Received request: frame_type=%lu seq=%lu x=%f y=%f yaw=%f",
         request.frame_type, request.seq, request.x, request.y, request.yaw);
```

---

#### 步骤2: 坐标转换（如果需要）

**转换逻辑**:

```cpp
req_frame new_request = request;

if (current_map_id_ == request.frame_type) {
    // 已经在目标地图，无需转换
    new_request = request;
} else {
    // 需要坐标转换：当前地图坐标 -> map1坐标 -> 目标地图坐标
    new_request = convertBetweenMaps(request, current_map_id_, request.frame_type);
}
```

**转换公式**（从mapA到mapB）:

1. **mapA -> map1**:
```
x_map1 = x_A * cos(θ_A) - y_A * sin(θ_A) + tx_A
y_map1 = x_A * sin(θ_A) + y_A * cos(θ_A) + ty_A
yaw_map1 = yaw_A + θ_A
```

2. **map1 -> mapB**:
```
x_B = (x_map1 - tx_B) * cos(θ_B) + (y_map1 - ty_B) * sin(θ_B)
y_B = -(x_map1 - tx_B) * sin(θ_B) + (y_map1 - ty_B) * cos(θ_B)
yaw_B = yaw_map1 - θ_B
```

**转换参数来源**: `floors.yaml` 配置文件

```yaml
maps:
  - id: 2
    tx_to_map1: 7.85      # map2相对map1的X平移
    ty_to_map1: 1.62      # map2相对map1的Y平移
    theta_to_map1: -1.55  # map2相对map1的旋转角（弧度）
```

#### 步骤6: 等待重定位完成

**等待信号**: `/map_to_odom_flag`

```cpp
std_msgs::Bool::ConstPtr relocalization_msg = 
    ros::topic::waitForMessage<std_msgs::Bool>("map_to_odom_flag", nh);

if (relocalization_msg && relocalization_msg->data) {
    reply.result = true;  // 切换成功
}
```

---

#### 步骤7: 发送回复给客户端

```cpp
replay_frame reply;
reply.seq = req_seq;
reply.result = true;  // 或 false

sendReplyOnSocket(client_fd, reply);
```

**回复数据结构**:
```cpp
struct replay_frame
{
    bool result;          // 切换是否成功
    unsigned long seq;    // 对应的请求序列号
};
```

### floors.yaml 配置文件

**位置**: `src/map_switch/config/floors.yaml`

```yaml
maps:
  - id: 1
    start_script: "/home/orangepi/slam_ws/src/map_switch/scripts/start_map1.sh"
    tx_to_map1: 0.0
    ty_to_map1: 0.0
    theta_to_map1: 0.0
    
  - id: 2
    start_script: "/home/orangepi/slam_ws/src/map_switch/scripts/start_map2.sh"
```

**参数含义**:
- `id`: 地图唯一标识符
- `start_script`: 启动该地图的脚本路径
- `tx_to_map1`: 该地图原点相对map1原点的X平移量（米）
- `ty_to_map1`: 该地图原点相对map1原点的Y平移量（米）
- `theta_to_map1`: 该地图坐标系相对map1坐标系的旋转角（弧度）

## 🚨 注意事项

### 1. 坐标系问题
- **重要**: 切换地图时，系统会自动进行坐标转换
- 转换参数必须在 `floors.yaml` 中正确配置
- 转换公式: 当前地图 → map1 → 目标地图

### 2. 初始位姿选择
**当前实现**: 使用实时坐标（第468行）
```cpp
launchMap(new_request.frame_type, new_request.seq, 
          new_request.x, new_request.y, new_request.yaw, client_fd);
```

**备选方案**: 使用固定坐标(0,0,0)（已注释）
```cpp
// launchMap(new_request.frame_type, new_request.seq, 0, 0, 0, client_fd);
```

**选择建议**:
- ✅ 使用实时坐标: 适合坐标稳定的环境
- ✅ 使用固定坐标: 适合坐标漂移严重的环境（如电梯内）

### 3. 地图文件命名
- map1: `scans_507_in.pcd` / `scans_507_in.yaml`
- map2: `scans_507_out.pcd` / `scans_507_out.yaml`
- 命名规则: `scans_XXX_YYY.pcd` / `scans_XXX_YYY.yaml`

### 4. 网络配置
- TCP服务器地址: `192.168.2.100`
- 地图切换端口: `6051`
- 客户端地址: `192.168.2.101`
