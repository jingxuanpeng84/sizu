# 地图切换问题排查指南

## 问题描述
SLAM端显示 `Map switch succeeded` 和 `send reply succeeded`，但前端一直显示"正在等待SLAM重定位"。

## 根本原因
**Socket接收超时时间过短**：原设置为5秒，但SLAM端需要约43秒完成地图切换，导致HTTP服务器后台线程在收到响应前就超时了。

## 已修复内容

### 1. 后端修复 (`httpcopy/main.cpp`)

#### 修复1: 延长接收超时时间
```cpp
// 修改前：
struct timeval tv;
tv.tv_sec = 5;  // ❌ 5秒太短
tv.tv_usec = 0;
setsockopt(sock, SOL_SOCKET, SO_RCVTIMEO, &tv, sizeof(tv));
setsockopt(sock, SOL_SOCKET, SO_SNDTIMEO, &tv, sizeof(tv));

// 修改后：
struct timeval tv_send;
tv_send.tv_sec = 5;   // ✅ 发送超时5秒
tv_send.tv_usec = 0;
setsockopt(sock, SOL_SOCKET, SO_SNDTIMEO, &tv_send, sizeof(tv_send));

struct timeval tv_recv;
tv_recv.tv_sec = 60;  // ✅ 接收超时60秒（等待SLAM完成）
tv_recv.tv_usec = 0;
setsockopt(sock, SOL_SOCKET, SO_RCVTIMEO, &tv_recv, sizeof(tv_recv));
```

#### 修复2: 增强调试日志
```cpp
// 后台线程增加详细日志
std::cout << "[地图切换] 后台线程开始等待SLAM响应..." << std::endl;

replay_frame reply;
memset(&reply, 0, sizeof(reply));  // 初始化结构体
ssize_t recv_len = recv(sock, &reply, sizeof(reply), 0);

std::cout << "[地图切换] recv返回: recv_len=" << recv_len 
          << ", result=" << (int)reply.result 
          << ", seq=" << reply.seq << std::endl;

if (recv_len == sizeof(reply) && reply.result) {
    std::cout << "[地图切换] ✓ SLAM端切换成功" << std::endl;
    g_map_switch_completed = true;
} else {
    std::cerr << "[地图切换] ✗ SLAM端切换失败或超时" << std::endl;
    g_map_switch_completed = false;
}
```

### 2. 前端优化 (`httpcopy/peizhi.html`)

#### 优化1: 增强状态检查逻辑
```javascript
async function waitForMapSwitchComplete() {
  const maxAttempts = 60;  // 60秒超时
  const interval = 1000;
  
  for (let i = 0; i < maxAttempts; i++) {
    const r = await fetch("/api/check_map_switch_status");
    if (r.ok) {
      const res = await r.json();
      console.log(`[${i + 1}/${maxAttempts}] 地图切换状态: ${res.status}`);
      
      if (res.status === "completed") {
        console.log("✓ 地图切换完成");
        return;
      } else if (res.status === "idle") {
        // 状态异常检测
        throw new Error("地图切换失败（状态异常：idle）");
      }
    }
    await new Promise(resolve => setTimeout(resolve, interval));
  }
  
  throw new Error("地图切换超时（超过60秒）");
}
```

## 故障排查

### 问题1: 前端仍然超时

**检查项**：
```bash
# 1. 确认HTTP服务器已重新编译和重启
ps aux | grep main

# 2. 检查HTTP服务器日志
# 应该看到 "recv返回: recv_len=16, result=1"
```

**可能原因**：
- HTTP服务器未重启，仍使用旧代码
- 网络连接问题

### 问题2: recv_len=0 或 recv_len=-1

**检查项**：
```bash
# 1. 确认SLAM端正在监听6051端口
netstat -tlnp | grep 6051

# 2. 测试TCP连接
telnet 192.168.110.205 6051
```

**可能原因**：
- SLAM端未启动或崩溃
- 端口被防火墙阻止
- IP地址错误

### 问题3: recv_len=16 但 result=0

**检查项**：
```bash
# 查看SLAM端日志，确认是否有错误
# 特别关注 "Map switch succeeded" 之前的错误信息
```

**可能原因**：
- SLAM端地图切换失败
- 重定位失败（`map_to_odom_flag` 话题未发布）
- 初始位姿发布失败

### 问题4: 结构体大小不匹配

**验证**：
```cpp
// 在HTTP服务器中添加调试代码
std::cout << "sizeof(replay_frame) = " << sizeof(replay_frame) << std::endl;
// 应该输出: sizeof(replay_frame) = 16 (bool=1字节 + padding + unsigned long=8字节)
```

## 性能优化建议

### 1. 减少地图切换时间
当前切换时间约43秒，主要耗时在：
- 等待 `waiting_for_initial_pose` 话题: ~34秒
- 发布初始位姿到重定位完成: ~6秒

**优化方向**：
- 检查为什么 `waiting_for_initial_pose` 需要34秒
- 优化SLAM节点启动速度

## 总结

**核心修复**：将socket接收超时从5秒延长到60秒

**关键点**：
1. ✅ SLAM端处理时间: ~43秒
2. ✅ Socket超时设置: 60秒（足够）
3. ✅ 前端轮询超时: 60秒（匹配）
4. ✅ 调试日志完善

**预期结果**：前端能正确检测到地图切换完成，对话框显示成功消息并自动关闭。
