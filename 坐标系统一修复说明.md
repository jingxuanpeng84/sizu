# 坐标系统一修复说明

## 问题描述

雷达在同一位置时，`map_switch` 和 `position_server_newdevice` 发送的世界坐标不一致。

## 根本原因

两个系统使用了**不同的坐标来源**：

### 1. `map_switch/tf_publish_node` 
- **数据来源**: TF变换
- **坐标系**: `/robot_body` → `/map`
- **代码位置**: `src/map_switch/src/tf_publish_node.cpp` 第389-396行
```cpp
listener.lookupTransform("/map", "/robot_body", ros::Time(0), TF);
```

### 2. `position_server_newdevice` (修复前)
- **数据来源**: `/Odometry` 话题
- **坐标系**: `/camera_init` → `/body`
- **代码位置**: `src/goalsender/position_server_newdevice.cpp` 第323-325行
```cpp
this->currentPositionData.x = msg->pose.pose.position.x;
this->currentPositionData.y = msg->pose.pose.position.y;
```

### TF树结构

```
/camera_init → /robot (静态TF)
/body → /robot_body (静态TF)
/pcd_map → /map (静态TF)
```

**问题**: `/Odometry` 发布的是 `camera_init` 坐标系下的位置，而 `tf_publish` 查询的是 `map` 坐标系下的位置，两者**不在同一坐标系**！

## 修复方案

让 `position_server_newdevice` 也使用 **TF变换** 获取位置，与 `map_switch` 保持一致。

### 修改内容

#### 1. 头文件 (`position_server_newdevice.hpp`)
- 添加 `tf::TransformListener tf_listener_`
- 添加 `source_frame_`, `target_frame_`, `use_tf_` 成员变量
- 添加 `tf_update_thread_` 和相关控制函数

#### 2. 实现文件 (`position_server_newdevice.cpp`)
- 添加 `startTFUpdateThread()`, `stopTFUpdateThread()`, `tfUpdateLoop()` 函数
- 在构造函数中根据 `use_tf_` 参数选择使用TF或Odometry
- TF更新线程以10Hz频率查询 `/map` → `/robot_body` 变换

#### 3. Launch文件 (`position_server_newdevice.launch`)
- 添加 `use_tf` 参数（默认true）
- 添加 `source_frame` 参数（默认 `/robot_body`）
- 添加 `target_frame` 参数（默认 `/map`）

### 配置参数

```xml
<!-- 使用TF模式（推荐） -->
<arg name="use_tf" default="true" />
<arg name="source_frame" default="/robot_body" />
<arg name="target_frame" default="/map" />

<!-- 使用Odometry模式（旧方式，不推荐） -->
<arg name="use_tf" default="false" />
<arg name="odom_topic" default="/Odometry" />
```

## 注意事项

1. **坐标系必须一致**: 确保 `source_frame` 和 `target_frame` 与 `map_switch` 配置相同
2. **TF树必须完整**: 确保 `/robot_body` 到 `/map` 的TF链路存在
3. **时间同步**: TF使用 `ros::Time(0)` 获取最新变换
4. **兼容性**: 保留了Odometry模式作为备选方案（设置 `use_tf=false`）
