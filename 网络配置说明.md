## 📋 端口配置表

### HTTP服务器端 (192.168.110.180)

| 服务 | 协议 | 端口 | 说明 |
|------|------|------|------|
| Web界面 | TCP | 8080 | 前端访问 http://192.168.110.180:8080 |
| UDP通信 | UDP | 8111 | 模块间通信（心跳等） |
| FTP服务 | TCP | 2121 | 地图文件上传 |

### SLAM端 (192.168.110.205)

| 服务 | 协议 | 端口 | 说明 |
|------|------|------|------|
| Position Server | UDP | 8080 | 接收位置查询请求 |
| 保存地图监听 | UDP | 9999 | 接收保存地图命令 |
| 上传地图监听 | UDP | 9998 | 接收上传地图命令 |

## 🔧 配置文件

### 1. HTTP服务器配置 (httpcopy/main.cpp)

```cpp
#define HTTP_PORT 8080
#define SERVER_IP "0.0.0.0"
#define UDP_PORT  8111
#define FTP_PORT  2121

// SLAM端地址
#define SLAM_IP   "192.168.110.205"
#define SLAM_PORT 8080
```

## 🔄 通信流程

### 获取位置数据流程

1. **前端请求**
   ```
   浏览器 → HTTP GET /api/get_position
   ```

2. **HTTP服务器处理**
   ```cpp
   // main.cpp
   SlamPose pose = request_slam_pose(SLAM_IP, SLAM_PORT);
   // 向 192.168.110.205:8080 发送UDP INQUIRY请求
   ```

3. **Position Server响应**
   ```cpp
   // position_server_newdevice.cpp
   // 接收INQUIRY请求
   // 从/Odometry话题获取当前位置
   // 发送RESPONSE响应（包含世界坐标和像素坐标）
   ```

4. **HTTP服务器返回**
   ```json
   {
     "x": 1.234,
     "y": 2.345,
     "yaw": 0.785,
     "px": 123,
     "py": 456
   }
   ```

## 🐛 故障排查

### 问题1: 超时无响应

**症状：**
```
❌ 错误: 超时 (3秒内未收到响应)
```

**排查步骤：**

1. **检查SLAM端服务是否运行**
   ```bash
   # 在SLAM端(192.168.110.205)执行
   ps aux | grep position_server
   netstat -ulnp | grep 8080
   ```

2. **检查网络连通性**
   ```bash
   # 在HTTP服务器端(192.168.110.180)执行
   ping 192.168.110.205
   ```

3. **检查防火墙**
   ```bash
   # 在SLAM端(192.168.110.205)执行
   sudo ufw status
   sudo ufw allow 8080/udp
   ```

4. **查看Position Server日志**
   ```bash
   # 在SLAM端查看ROS日志
   rosnode info /position_server_node
   rostopic echo /Odometry -n 1
   ```

### 问题2: 位置数据全为0

**症状：**
```
⚠️  警告: 位置数据全为0
```

**原因：**
- /Odometry话题未发布
- project.launch未启动
- 定位模块未初始化完成

**解决方法：**
```bash
# 在SLAM端检查
rostopic list | grep Odometry
rostopic hz /Odometry
rostopic echo /Odometry -n 1
```

### 问题3: 端口冲突

**症状：**
```
bind: Address already in use
```

**解决方法：**
```bash
# 查找占用端口的进程
netstat -ulnp | grep 8080
# 或
lsof -i UDP:8080

# 终止占用进程
kill -9 <PID>
```

## 🔒 安全建议

1. **防火墙配置**
   - 仅允许192.168.110.180访问SLAM端的8080端口
   - 限制其他IP访问

2. **网络隔离**
   - 将机器人控制网络与办公网络隔离
   - 使用独立的VLAN

3. **访问控制**
   - 在position_server中添加IP白名单验证
   - 记录所有访问日志
